Scope Creep

During the project, the scope focus might require additional features or team members want to devise further functionality to complete the app. Therefore, it would take more time to complete the deliverables and lessen the productivity of working on tasks that are out of scope.

Solution

To avoid this, the whole team will do research in depth on the requirements needed for the project implementation. Besides, The project manager will consistently keep track of the process to evaluate the feasibility if changes are made to the project.

Timeline Risk

When working on the report, some sections might require extensive knowledge of finance which most of the team members are not familiar with. This could lengthen the time needed to finish the report on time, especially when the deadlines of other courses might overlap each other.

Solution

When deciding on the topic, we agreed to delve more into the financial industry for research material and field insight. Additionally, the team agreed to schedule their progress one week prior to the deadline to resolve any problems that might arise.

Testing Risk

The project failed the marketing process or the products are not engaging enough that there is not enough constructive feedback from users for the team to evaluate. The further development process might not be productive during the testing period.

Solution

Team members themselves will seek feedback from colleagues, mentors, and acquaintances to gain the required number of users testing the product, facilitating the advancement of the project.

Technical difficulties

In the team, some members had no previous exposure to programming so for the project prototype, using HTML, CSS, and Figma might become a hurdle.

Solution

The team members are assigned to self-study basic web programming in the first few weeks to understand the structure of the project. Besides, coding tasks can be allocated to members who are more experienced in the field.

Git Conflicts

Pull and merge conflict happens when there are different changes in local and remote branches. This could cause incompletion of the website where sections are not merged.

Solution

Each team member is suggested to leave comments for changes made to the branches and minimize the number of branches.

Data loss

If group members might accidentally delete the report, code file or Github server crash can cause the project data to be lost.

Solution

To prevent this, we saved a backup file of the report on Google Drive and keep a record of each member's codes.

User Information Security

When using our website, users might fall victim to cyber threats such as viruses, malware, and phishing websites, from which hackers can get access to user personal information (bank accounts, transaction details, …) inside the app.

Solution

User data needs to be encrypted to secure customers' privacy as well as avoid any threats from viewing the database once they bypass the server. If users log in from a different device, the website will require multi-factor authentication to verify their credentials via registered email or SMS. For better protection of user information, the data sent to the server will be encrypted to avoid attacks from the database.
